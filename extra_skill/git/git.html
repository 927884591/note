# git && github的基本使用

# 使用git的基本流程

  1. 工作区与暂存区的交互；
  2. 暂存区与本地仓库（分支）上的交互；
  3. 本地仓库与远程仓库的交互。

# 图形化安装

git官方网站 (opens new window)

也可以使用

使用brew install git

# 配置

git的选项命令都是使用--

$ git config --global user.name test
$ git config --global user.email test@qq.com
1
2

这两个是最重要的命令，

# 列出git的配置

使用的是git config 命令

git config --list //列出当前的配置选项
git config --local --list //列出repository的配置
git config --global --list //列出全局的配置选项
git config --system --list //列出系统的配置选项

# git的帮助菜单

git config help //这个是完整的帮助文件，会以网页的形式打开，且文件存放到本地中
git config -h //在终端直接显示简明的帮助信息
1

# git获取仓库的两种方式

  1. 把本地的文件初始化成git仓库文件，让git来帮我们管理文件
  2. 直接克隆一个git仓库到本地
git init //初始化本地文件，让git来帮我们管理仓库
1

# 基本使用

git init //初始化当前目录，它会给当前目录加一个.git的文件夹
git clone <地址>    //克隆指定的项目地址到当前目录中
git add -all //将仓库中的所有文件添加到暂存区
git commit  //暂存区提交到本地仓库
git status //查看git的状态。
git push  //推送到远程仓库中
git remote -v    //查看远程仓库的信息

# git移除文件

  1. 移除工作区和git仓库的文件

    git rm -f 文件名
    1

  2. 移除git仓库的文件

    git rm --cached 文件名
    1

# git切换版本

git rest --hard <commitID> //回退到指定的版本
1

# git的多人协助

多人协助是git重要的功能之一。

git branch -a 	//显示所有的本地分支和远程分支
git branch -r  	//显示远程仓库的分支
git checkout -b 分支名   //新建本地分支并且切换到该分支中
git branch -d 分支名    //如果已经合并了分支，就可以使用这条命令来删除该分支
git merge 分支名   //合并该分支到本分支中
git push origin -d 分支名	 //删除远程分支
1

git中的origin指的是使用远程仓库

# git中的后悔药

checkout的本质是用git仓库的文件替换到工作区中，因为在呢没有提交前，仓库的文件都是旧的，所以撤销会还原仓库的版本

git checkout --文件名 //指定某个文件测小工作区的操作
git reset HEAD 文件名 或 . //从暂存区移除当前文件
1
2

# 生成密钥

为什么要生成密钥，主要是为了方便推送到远程仓库，

使用git生成会得到

  • 私钥
  • 公钥

生成密钥命令

ssh-keygen -t rsa -C "8888@163.com" //8888@163.com是你自己注册GitHub的邮箱
1

然后到密钥的目录中找出公钥文件复制到github中即可。
