# 数组类型

数组类型比较特殊，因为我们虽然已经确定了他说一个Array的类型，但数组里的元素我们没有确定类型

第一种写法

const name: Array<string> = []   //定义一个数组，并且数组里的类型是string的类型  <>里填的叫泛型
name.push("zhangsan"),
name.push(123)  //报错，不能push一个为number的类型

# void类型

void类型是没有返回值的，一般我们不需要写。如果函数中你没有设置返回值的话默认是一个undefind。这时候判断就是为void类型

function foo(num: number):void{
  console.log(num)
}    //这个void也可以不写

# 元组类型（tuple）

由于数组类型的弊端，一般默认的数组存放的元素可以是多种类型，而这多种类型就会导致不安全，所以为了保证数组的元素的类型的一致性，所以有了元组类型

# 函数重载

函数重载指的是函数名称相同，函数的参数的类型不同或个数的不同。

function add(num1: number,num2:number);  //1
function add(num1: string,num2: string);  //2
function add(num1: any,num2: any){   //这里写类型一定要包含重载的类型，需要写的宽裕一点
  return num1 + num2
}
add(10,20)   //会调用第一个重载函数。 typescript会根据参数的类型匹配重载函数
add("zhansan","lisi")   //调用第二个

# 类型参数化

类型参数化指的是在写函数中我们不定义参数的类型，而是需要调用者来告诉我们你需要什么类型

function sum<T>(num1: T,num2: T){   //<T>为传过来的类型参数 
  return num1+num2 //这里会报错，因为这边是不知道这个T是不是可以相加的类型
}
sum<number>(20,30)   //调用者把number类型传过去
