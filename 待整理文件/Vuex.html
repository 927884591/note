# Vuex

# 简介

vuex给vue组件控制状态和监控状态的插件，相当于一个管理变量的一个管家，这有什么好处呢？如果组件与组件之间的关系离的远而不仅仅只是子传父或者父传子，这样的传值就会很复杂，我们可以把想共享的变量放到一个公共的对象中，如何组件都可以访问的到的，我们就能很好的解决这个问题，所以有了vuex。

vuex有5个核心

  • state
  • view
  • actions
  • getters
  • Mutations

数据驱动视图，而视图交互产生的事件又驱动数据

//index.js
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)
const store = new Vuex.Store({
  state:{
    //需要共享的变量放这里
  }，
  mutations:{
  	//需要处理的变量都要经过这个，否则devtools工具无法监控
  	//这个会默认传一个state过来
	},
  actions:{
		
  },
  modules:{
    
  },
  getters:{
    //类似于computed属性，里面定义是一个函数，可以当作一个属性去使用
    //在这里写方法都会默认传两个参数过来state和getters本身
  }
})
export default store			//把vuex导出去

# Actions

![image-20220303120105482](/Users/happy/Library/Application Support/typora-user-images/image-20220303120105482.png)

如上图所示，只有经过异步操作我们才需要使用action，如果是同步的话就直接使用mutations就可以了

//./store/index.js文件
import { createStore } from "vuex"
const store = createStore({
  state(){
    return{
      //这里写全局状态
      count:0
    }
  },
  mutations:{
    //同步操作状态的话需要经过mutations来改变state
    increment(state){
      state.count++
    }
  },
  actions:{
    //异步操作状态的话需要经过action来调用mutation的方法然后再改变state
    incrementAction(context){
      context.commit("increment")
    }
  }
})

# 单一状态树

vuex建议使用单一状态树，意思就是把全部组件的state保存到一个store中。这个可能会有什么坏处呢，就是所有的组件的状态都放在一个store里面就会导致这个store很难去维护。

# 解决难维护的问题

我们这个提取每个组件的状态，然后在导入到store里，就可以分模块的取管理我们组件的状态

# moduel

每一个模块都有vuex的核心。
